### Product changes

- **Workspaces launch** -- Organize your OpenRouter projects into separate environments, each with its own settings. See the [blog post](https://openrouter.ai/announcements/introducing-workspaces) and [docs](/guides/features/workspaces).
  - Scoped API keys, guardrails, routing defaults, and BYOK provider keys per workspace
  - Per-workspace observability with activity filtering, I/O logging, and usage charts
  - Workspaces CRUD API with bulk membership management
- **Activity Explorer enhancements** -- Added user dimension filtering, app name resolution, and context length bucket dimension to the [Activity Explorer](https://openrouter.ai/activity).
- **Server tools UI redesign** -- Redesigned the server tools interface in the [Playground](https://openrouter.ai/playground) with a unified dropdown, hover sub-panels, and smart image generation routing.
- **Web search engine options** -- Added Firecrawl and Parallel as web search engine options in the [Playground](https://openrouter.ai/playground).
- **Web search pricing display** -- Gemini endpoints with native web search support now show web search display pricing on model pages.
- **`web_fetch` in chat** -- The `web_fetch` server tool is now available in the web chat UI.
- **Image output token pricing** -- Image output tokens are now shown in display pricing for supported models.
- **Logs page improvements** -- Models are now grouped by provider in the sessions tab, with a redesigned generation detail side panel.
- **Tools docs link** -- Added a docs link icon next to the Tools title in the dropdown.
